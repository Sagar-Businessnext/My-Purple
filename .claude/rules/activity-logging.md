# Activity Logging Rule

Every significant action must be logged to `project/.claude/log.md`. This creates an audit trail for the project and helps team members understand what happened and why.

## When to Log

- Every command execution (`/bnac-build-fix`, `/bnac-code-review`, `/bnac-plan`, etc.)
- File creation or significant modification
- Context files read at session start
- Milestone task completion or progress
- Quality gate results
- Build/test/lint outcomes
- Errors encountered and how they were resolved

## Log Format

Append entries to the end of `project/.claude/log.md`:

```markdown
## [YYYY-MM-DD HH:mm] <action-type>: <brief description>

- **Command:** /command-name (if applicable)
- **Context read:** CLAUDE.md, SUMMARY.md, milestone-status.md
- **Files modified:** list of files changed
- **Result:** success | failure (with details)
- **Notes:** any relevant context or decisions made
```

## Action Types

| Type | When |
|------|------|
| `session-start` | Beginning of new conversation |
| `command` | Executing a BNAC command |
| `feature` | Implementing a feature |
| `fix` | Fixing a bug or build error |
| `review` | Code review completed |
| `milestone` | Milestone task progress |
| `quality-gate` | Quality gate run |

## Auto-Append Behavior

All agents MUST append to `project/.claude/log.md` automatically after every command execution. This is not optional — the `bnac-developer`, `bnac-reviewer`, planner agents (`bnac-planner`, `bnac-phase-planner`, `bnac-milestone-planner`, `bnac-task-planner`), `bnac-milestone-tracker`, and `bnac-quality-gate` agents all log their actions.

**How to append:** Use **Edit** to add a new entry at the end of `log.md`. Never overwrite existing entries.

## Rules

1. **Log immediately** — Don't batch logs. Log as you go.
2. **Be specific** — "Modified 3 files" is useless. List the files.
3. **Include outcomes** — Did the build pass? Did tests fail? How many errors?
4. **Log failures too** — Failed attempts are valuable context.
5. **Don't log trivial reads** — Reading a file for context doesn't need a log entry unless it informed a decision.

## If log.md Doesn't Exist

If `project/.claude/log.md` doesn't exist, create it with this header:

```markdown
# Activity Log

> Auto-generated by BNAC agents. Do not edit manually.

---
```

Then append your entry.
