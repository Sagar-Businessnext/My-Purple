------------------------------------------------------------------------------
 PART 0 — ONE-TIME PREREQUISITES (skip any you already installed)
------------------------------------------------------------------------------
Install these once, from their websites, then reboot if asked:

  1. Python 3.12         https://www.python.org/downloads/   (tick "Add to PATH")
  2. Ollama              https://ollama.com/download
  3. PostgreSQL 16 + pgvector
                         https://www.postgresql.org/download/windows/
                         pgvector: https://github.com/pgvector/pgvector (or use
                         the "StackBuilder" extension, or EDB's bundle)
  4. ffmpeg              https://www.gyan.dev/ffmpeg/builds/  (add bin\ to PATH)
                         (needed for microphone / Whisper audio decoding)
  5. Node.js LTS         https://nodejs.org   (only if you want the desktop UI)

Confirm they're on PATH (each should print a version):
  > python --version
  > ollama --version
  > psql --version
  > ffmpeg -version


------------------------------------------------------------------------------
 PART 1 — SETUP (one command, idempotent — safe to re-run)
------------------------------------------------------------------------------
  > cd D:\Purple
  > powershell -ExecutionPolicy Bypass -File setup.ps1

This creates the virtual env, installs Purple, pulls the Ollama models
(qwen2.5:14b + nomic-embed-text), creates the Postgres DB + pgvector, downloads
the Kokoro voice, copies .env, and finishes by running the self-check.

Optional extras (install only the ones you want to test):
  > .\.venv\Scripts\Activate.ps1
  > pip install -e ".[speaker]"        # voice access gate (only your voice talks to her)
  > pip install -e ".[google]"         # Gmail + Calendar
  > pip install -e ".[observability]"  # Prometheus /metrics
  > python -m playwright install chromium   # browser control (setup.ps1 already tries this)


------------------------------------------------------------------------------
 PART 2 — CONFIGURE (open D:\Purple\.env in Notepad)
------------------------------------------------------------------------------
  > notepad D:\Purple\.env

Set at least your city so weather + greeting work (no API key needed):
      PURPLE_WEATHER_LOCATION=Pune          (use your city)

Good defaults already set — change only if you want:
      PURPLE_ENABLE_GREETING=true           time-aware hello on startup
      PURPLE_BRIEFING_HOUR=7                 daily briefing time (24h)
      PURPLE_OBSERVE_DEFAULT=false           screen observation stays OFF until you ask
      PURPLE_OBSERVE_AUTO_OFF_HOURS=4        observation auto-stops after 4h
      PURPLE_WHISPER_DEVICE=cuda             set to "cpu" if you have no NVIDIA GPU
      PURPLE_REQUIRE_CONFIRMATION=true       risky actions always ask first (keep true)

Save and close Notepad.

------------------------------------------------------------------------------
 PART 4 — START PURPLE
------------------------------------------------------------------------------
Leave this window open — it's the running server.
  > cd D:\Purple
  > .\.venv\Scripts\Activate.ps1
  > purple

(Equivalent: python -m purple.run)
On startup the built-in web UI opens automatically in your browser at
http://127.0.0.1:8765/ui  (chat + activity feed + Observe toggle). If it doesn't
open, just visit that URL. To stop it auto-opening: PURPLE_OPEN_UI_ON_START=false.
You'll also get a GREETING (a toast + a line in the feed, spoken if your speakers
are on). On the very first run you'll ALSO get a one-time "catch-up" briefing,
because no previous briefing is on record yet - that's expected; after that it
only catches up a missed briefing on boot.


------------------------------------------------------------------------------
 PART 5 — QUICK SMOKE TEST (open a SECOND PowerShell window; server keeps running)
------------------------------------------------------------------------------
These hit the API directly, so they work even without the desktop UI.

Health (expect ok:true, ollama:true, database:true):
  > Invoke-RestMethod http://127.0.0.1:8765/health | ConvertTo-Json -Depth 5

Talk to her:
  > Invoke-RestMethod -Method Post http://127.0.0.1:8765/chat -ContentType 'application/json' -Body '{"message":"Hi Purple, who are you?"}'

Weather (after setting your city in .env):
  > Invoke-RestMethod -Method Post http://127.0.0.1:8765/chat -ContentType 'application/json' -Body '{"message":"What is the weather right now?"}'

News (real headlines, fetched live):
  > Invoke-RestMethod -Method Post http://127.0.0.1:8765/chat -ContentType 'application/json' -Body '{"message":"Give me today news"}'

Full briefing on demand:
  > Invoke-RestMethod -Method Post http://127.0.0.1:8765/chat -ContentType 'application/json' -Body '{"message":"Give me my briefing"}'

Screen observation toggle (privacy feature — off by default):
  > Invoke-RestMethod http://127.0.0.1:8765/observe                      # status
  > Invoke-RestMethod -Method Post http://127.0.0.1:8765/observe -ContentType 'application/json' -Body '{"on":true}'   # start
  > Invoke-RestMethod -Method Post http://127.0.0.1:8765/observe -ContentType 'application/json' -Body '{"on":false}'  # stop
  (Or just tell her in chat: "start observing"  /  "stop observing".)

What improvements she's noticed (self-suggestions + config nudges):
  > Invoke-RestMethod http://127.0.0.1:8765/suggestions | ConvertTo-Json

------------------------------------------------------------------------------
 PART 7 — THE UI  (one React app; setup.ps1 builds it with Vite)
------------------------------------------------------------------------------
Web UI (default): the full React app (Chat / Monitor / Automations / Memory /
Missions / Settings, with the Observe toggle). setup.ps1 installs Node and builds
it; Purple serves it and opens it on startup at:
  http://127.0.0.1:8765/ui/
If you skipped the build, do it manually:
  > cd D:\My-Purple\frontend
  > npm install
  > npm run build        (outputs to ..\src\purple\web -> served at /ui/)
For live UI development with hot reload:  npm run dev   (http://localhost:5173)

Native desktop app (optional, same app in its own window): needs Node + Rust.
  > powershell -ExecutionPolicy Bypass -File scripts\build_tauri.ps1
First run installs Rust, scaffolds the Tauri shell, and opens the window. The
first build is slow (compiles Rust). For an installer: cd frontend; npm run tauri build


------------------------------------------------------------------------------
 PART 8 — TROUBLESHOOTING (most likely issues first)
------------------------------------------------------------------------------
* selfcheck says Ollama down:
      > ollama serve            (run in its own window)
      > ollama pull qwen2.5:14b-instruct-q4_K_M
* selfcheck says database/pgvector down:
      Make sure PostgreSQL service is running, then:
      > psql -U postgres -h 127.0.0.1 -d purple -c "CREATE EXTENSION IF NOT EXISTS vector;"
* It's very slow / out of memory (you have 16GB):
      qwen-14b is heavy. In .env lower the load:
      PURPLE_WHISPER_MODEL=small      and if needed try a smaller LLM:
      PURPLE_LLM_MODEL=qwen2.5:7b-instruct-q4_K_M   (then: ollama pull that model)
* No NVIDIA GPU:
      In .env set PURPLE_WHISPER_DEVICE=cpu and PURPLE_WHISPER_COMPUTE=int8
* Microphone / audio errors:
      Make sure ffmpeg is on PATH; check Windows mic permissions.
* "playwright" / browser errors:
      > python -m playwright install chromium
* See what happened:
      > Invoke-RestMethod "http://127.0.0.1:8765/logs?n=80"
      or open D:\Purple\logs\purple.log
